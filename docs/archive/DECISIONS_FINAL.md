## ✅ 최종 결정 사항 요약

### 🔴 Phase 1 MVP 핵심 변경사항

**중요**: 페널티 기능을 **완전히 폐기**하고, 순수한 "시간 관리 보조 도구"로 방향 전환

```
AS-IS (원래 계획):
- 역산 스케줄링 + 실시간 교통 + 금전적 페널티

TO-BE (최종 확정):
- 역산 스케줄링 + 실시간 교통 + 시각적 알람 + 대중교통 + 홈 위젯
- 페널티 기능 완전 폐기 (비즈니스 모델 변경)
```

**폐기 이유**:
- 앱스토어 심사 리스크 회피
- 법적 규제 복잡성 제거
- 긍정적 사용자 경험 집중
- 빠른 시장 진입 우선

## 🎯 Phase 1 MVP 재정의 (페널티 제외)

### 핵심 가치 제안
```
"AI가 계산한 정확한 출발 시간으로 절대 안 늦는 습관 만들기"

핵심 기능만 집중:
1. 역산 스케줄링 (도착 시간 → 출발 시간 자동 계산)
2. 실시간 교통 반영 (네이버 API)
3. 시각적 카운트다운 (대시보드)
4. 준비 시간 설정
5. 일정 관리 (데일리/먼슬리 뷰)
6. ⭐ 대중교통 지원 (버스/지하철 실시간 정보)
7. ⭐ 홈 위젯 (Android/iOS 필수)
```

```
❌ 금전적 페널티 시스템 (완전 폐기)

❌ 지오펜싱 출발 감지 (폐기)
   - 이유: 패널티 시스템과 연계 기능

❌ Stripe/토스페이먼츠 외부 PG 연동 (폐기)
   - 이유: 앱스토어 IAP 정책 위반

❌ 억울한 패널티 소명 프로세스 (폐기)
   - 이유: 패널티 기능 제거로 불필요
```
