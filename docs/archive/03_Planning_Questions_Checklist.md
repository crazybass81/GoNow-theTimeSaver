## 📋 사용 방법 / How to Use

이 문서는 'Go Now' 프로젝트의 **핵심 의사결정 사항**을 체계적으로 정리하기 위한 체크리스트입니다.

**체크 표시 규칙**:
- ✅ 결정 완료
- 🔄 검토 중
- ❌ 결정 보류
- ⚠️ 리스크 있음 (추가 검증 필요)

**우선순위**:
- 🔴 **P0 (Critical)**: Phase 1 개발 전 필수 결정
- 🟡 **P1 (High)**: Phase 2 개발 전 필수 결정
- 🟢 **P2 (Medium)**: Phase 3 또는 이후 결정 가능

## A. 비즈니스 모델 및 규제 / Business Model & Regulation

### A1. 페널티 수익 귀속 여부 🔴 P0

**질문**:
사용자가 실패하여 차감된 돈(예: 10,000원)은 어떻게 배분할 것인가?

**옵션**:

| 옵션 | 설명 | 장점 | 단점 | 상태 |
|------|------|------|------|------|
| A. 100% 회사 수익 | 전액을 회사 매출로 계상 | 수익 극대화 | 도덕적 비난 리스크, "앱이 일부러 늦게 알림" 의심 | [ ] |
| B. 일부 기부 (30~50%) | 자선단체에 일부 기부 | 도덕적 명분 확보 | 수익 감소 | [ ] |
| C. 사용자 풀 (Jackpot) | 성공자들끼리 실패자 페널티 나눠가짐 | 게임화 강화 | 사행성 규제 저촉 가능성 | [ ] |
| D. 플랫폼 수수료만 (20%) | 나머지는 기부 또는 보험 펀드 | 공정성 인정 | 최저 수익 모델 | [ ] |

**결정 사항**:
```
[ ] 최종 선택: _____________
[ ] 앱스토어 가이드라인 확인 완료
[ ] 법률 자문 완료
```

**추가 고려사항**:
- Forfeit 사례: 실패 시 자선단체 기부 (사용자 선택)
- StickK 사례: 20% 수수료 + 나머지는 사용자 지정 수취인

### A3. PG사 선정 🔴 P0

**질문**:
결제 게이트웨이는 어떤 서비스를 사용할 것인가?

| PG사 | 장점 | 단점 | 적합성 | 상태 |
|------|------|------|--------|------|
| **Stripe** | 글로벌 표준, API 우수, PaymentIntent 지원 | 한국 카드사 수수료 높음 | 글로벌 확장 시 필수 | [ ] |
| **토스페이먼츠** | 한국 최적화, 간편결제 지원 | 해외 확장 제한적 | 한국 전용 서비스 시 | [ ] |
| **나이스페이** | 기업 신뢰도 높음 | API 복잡도 높음 | 대기업 제휴 시 | [ ] |

- "벌금" 성격의 자동 결제를 PG사가 허용하는가?
- Stripe는 PaymentIntent의 `capture_method: manual` 지원 → ✅ 적합
- 국내 PG는 이러한 기능이 제한적일 수 있음 → ⚠️ 사전 확인 필요

**결정 사항**:
```
[ ] Phase 1 (MVP): _____________ 사용
[ ] Phase 3 (글로벌): _____________ 추가 연동
[ ] PG사와 사전 협의 완료 (페널티 정책 승인)
```
